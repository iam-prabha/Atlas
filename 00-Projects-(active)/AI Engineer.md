# AI Engineer Roadmap
> Based on analysis of 1,000+ job descriptions · [AI Shipping Labs](https://aishippinglabs.com/ai-engineer-learning-path/)

---

## Stage 1 — Python & LLM Foundations
**Weeks 1–3 · Python appears in 82.5% of job postings**

- Python fluency — data structures, async, APIs
- How LLMs work — tokenization, context windows, costs
- Calling OpenAI / Anthropic APIs
- Prompt engineering basics

---

## Stage 2 — RAG & Retrieval Systems
**Weeks 4–7 · RAG is the #1 GenAI skill at 35.9% of jobs**

- Embeddings and semantic search
- Vector databases — Pinecone, Weaviate, pgvector
- Chunking strategies and document processing
- Hybrid retrieval and re-ranking

---

## Stage 3 — Agents & Workflows
**Weeks 8–11 · Agentic workflows in 14.4% of jobs**

- Tool use and function calling
- Multi-step reasoning and state management
- LangChain / LangGraph / LlamaIndex
- Error handling and fallback strategies

---

## Stage 4 — Production & Deployment
**Weeks 12–15 · Docker 31% · CI/CD 29.3% · AWS 40.1%**

- FastAPI for serving AI endpoints
- Docker and containerization
- CI/CD pipelines
- Cloud deployment — AWS / GCP / Azure

---

## Stage 5 — Evaluation & Observability
**Weeks 16–19 · Critical but often overlooked**

- Building evaluation datasets and test sets
- LLM-as-judge patterns
- Monitoring drift and output quality
- Guardrails and safety checks

---

## Stage 6 — Scale & Specialization
**Weeks 20–24 · Kubernetes 29.1% · Fine-tuning 8.5%**

- Kubernetes and infrastructure at scale
- Inference optimization — latency, cost, throughput
- Fine-tuning for domain-specific tasks
- Security, compliance, and responsible AI

## Portfolio Projects

### 01 · Production RAG System *(Foundational)*
Build a production-ready RAG system over a real dataset. Include chunking strategy, retrieval method, evaluation plan, and deployment architecture.

**Skills:** RAG · Vector DB · Python · Docker · Evaluation

---

### 02 · Multi-Step AI Agent *(Intermediate)*
Build an agent that automates a repetitive workflow in your industry. Include data ingestion, tool use, logging, and error handling.

**Skills:** Agents · LLM APIs · Python · LangChain/LangGraph

---

### 03 · Cloud-Native AI Deployment *(Intermediate)*
Design a cloud-native deployment architecture for an LLM-powered service including Docker, CI/CD, monitoring, and scaling.

**Skills:** Docker · CI/CD · AWS/GCP/Azure · Kubernetes · Monitoring

---

### 04 · AI for Risk or Compliance *(Advanced)*
Build an AI system that assists with risk or compliance decisions in a regulated industry. Include architecture, evaluation, and monitoring.

**Skills:** LLM Integration · Evaluation · RAG · Production Ops

---

## Core Responsibilities

- **Build AI systems** — end-to-end LLM-powered apps: RAG, agents, prompt pipelines
- **Productionize AI** — APIs, cloud deployment, monitoring, failure handling
- **Evaluation & quality** — pipelines, observability, guardrails, reliability standards
- **Use provider APIs** — rate limits, cost control, retries, streaming, fallbacks
- **RAG & retrieval** — vector search, hybrid retrieval, metadata filtering, re-ranking
- **Agents & workflows** — tool calling, state management, multi-step reasoning
